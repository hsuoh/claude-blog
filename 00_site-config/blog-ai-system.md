## 前提：このサイトの設計思想

- **サイト名**: 薬剤師キャリアナビ（pharmirai.com）
- **コンセプト**: 転職を煽らない。データから判断させる。
- **最終ゴール**: 収益構造を理解し、合理的に転職メリットがある薬剤師を転職サイトにつなぐ
- **ターゲット**: 20〜30代の病院・調剤薬局・ドラッグストア薬剤師
- **運営体制**: PM（あなた）＋ 記事制作パートナー＋ Claude（AI担当）
- **参考設計**: リベ大ブログ教室（収益化設計）× トピッククラスター戦略（SEO構造）

## ① AIチーム構成

| 役割 | 担当 | 主な責務 |
|---|---|---|
| **PM** | あなた（人間） | 方針決定・記事承認・公開判断・最終品質管理 |
| **リサーチAI** | Claude | KW調査・競合分析・ニーズ収集・データ収集 |
| **ライターAI** | Claude | 構成作成・本文執筆・見出し設計・ディスクリプション生成 |
| **SEO設計AI** | Claude | 内部リンク設計・カテゴリ分類・収益導線設計 |
| **データ分析AI** | Claude | 決算データ整理・調剤報酬改定解説・業界ニュースまとめ |
| **WP入稿担当** | パートナー（人間） | WP入稿・装飾・アイキャッチ作成・公開設定 |

**運用の基本原則**:
1. 記事1本につき「KW選定 → リサーチ → 構成 → 執筆 → 内部リンク設計」をClaudeが一気通貫で行う
2. PMが方向性・KW・タイトルを承認してから執筆に入る
3. 執筆後はPMが一次レビュー → パートナーが入稿

## ② 役割定義

### PM（プロジェクトマネージャー）の役割
- 月次の記事テーマ優先順位を決定する
- AIが出した構成案・タイトル案を承認・修正する
- 現役薬剤師として専門的な事実確認を行う
- 転職サイトアフィリエイトの掲載タイミングを判断する

### Claude（リサーチAI）の役割
- ラッコキーワードのサジェスト結果をもとにKW候補を整理する
- 調剤報酬改定の厚労省資料・業界ニュースを要約する
- 大手薬局の決算情報を整理して記事素材を作る
- 競合上位記事の見出し構成を分析してギャップを見つける

### Claude（ライターAI）の役割
- 読者が「転職すべきか・残るべきか・様子見か」を判断できる記事を書く
- 転職を煽らず、データと構造で読者が自己判断できる文章にする
- 吹き出し・ボックス・比較表を適切に使い、読みやすい構成にする
- 各記事に必ずディスクリプション・タイトルタグ候補を3案出す

## ③ 記事候補選定手順

### 記事の3分類（リベ大方式）

| 分類 | 役割 | 対応フォルダ |
|---|---|---|
| **まとめ記事（ピラー）** | テーマ全体を網羅・詳細記事への入口 | `01_pillar` |
| **集客記事（クラスター）** | ロングテールKWで流入を集める・まとめ記事に誘導 | `02_articles` / `05_stock` |
| **キラーページ（収益）** | 転職サイト比較・ランキング・直接的なCTA | `02_articles`（収益特化） |

データ記事（`03_data`）はまとめ記事・集客記事の両方を支える素材として機能する。

### 選定手順

```
STEP 1: ターゲットの悩みを3軸で整理する
  └ ① お金の悩み（給与・ボーナス・昇給が止まった）
  └ ② 職場の悩み（人間関係・残業・待遇が悪い）
  └ ③ 将来の不安（調剤報酬改定・薬剤師余剰・AI）

STEP 2: 各悩みをキーワードに変換する
  └ 「薬剤師 給料 下がる」「調剤薬局 将来性」など

STEP 3: 記事分類とフォルダを決める
  └ まとめ記事（01）/ 集客記事（02・05）/ データ記事（03）/ キラー（02）

STEP 4: 不要な記事を明確に除外する
  └ 収益ゴール（転職サービス登録）と無関係な記事は書かない
  └ 例：「薬剤師 趣味」「薬剤師 勉強法」などは対象外

STEP 5: PMが優先順位を決定して月次計画に入れる
```

**選定の判断基準（優先度順）**:
1. 転職判断・給与・収益構造に直接関連するテーマ
2. 検索ボリューム 100〜3,000（競合が弱い帯を狙う）
3. 現役薬剤師として専門性で差別化できるテーマ
4. 調剤報酬改定・決算データで裏付けできるテーマ

## ④ キーワード選定手順

**使用ツール**:
- [ラッコキーワード](https://rakkokeyword.com)：サジェスト取得・見出し抽出・Q&A取得（要ログイン）
- [Googleキーワードプランナー](https://ads.google.com/home/tools/keyword-planner/)：検索ボリューム確認（無料）
- Google検索：競合・検索意図の確認（無料）

```
STEP 1: シードKWを決める
  └ site-profile.mdの「主要SEOキーワード」リストを起点にする
  └ 例：「薬剤師 給料」「病院薬剤師 年収」「調剤薬局 将来性」

STEP 2: ラッコキーワードでサジェストを展開する
  └ rakkokeyword.com にログインしてシードKWを入力
  └ 「Googleサジェスト」で検索 → 一覧をコピー
  └ ラッコの「見出し抽出」で上位記事の構成も取得する
  └ ラッコの「Q&A取得」でYahoo!知恵袋の質問も確認する
  └ 取得したデータを付録Bのテンプレを使ってClaudeに渡す

STEP 3: 検索ボリュームを確認する
  └ Googleキーワードプランナーにサジェスト一覧を貼り付けて確認
  └ 狙い目：月間100〜3,000（新規サイトは100〜1,000から）
  └ 0・10未満は基本スキップ

STEP 4: Googleで実際に検索して競合を確認する
  └ 上位10件のサイトタイプを確認する
     ├ 個人ブログ・中小メディアが多い → 勝てる（◎）
     ├ 大手転職サイト・求人サイトが多い → 難しい（△）
     └ 厚労省・公式機関・大手企業ばかり → 避ける（✕）
  └ ライバル記事のタイトルにKWが含まれているか確認
  └ ライバル記事の内容・文字数・網羅性を確認
  └ 自分が差別化できる切り口があるか判断する

STEP 5: 検索意図を分類してフォルダを決める
  └ 情報収集型（know）→ 05_stock / 03_data
  └ 比較・判断型（do） → 02_articles / 01_pillar
  └ 転職意欲型（go）  → 02_articles（収益記事）

STEP 6: タイトルKWを確定してPMが承認する
  └ Claudeにタイトル案3つを出させてPMが選ぶ
```

**KWの選定ルール**:
- 1記事 = メインKW1つだけ（カニバリ防止）
- 共起語・関連語は本文中に自然に含める
- 「薬剤師」「調剤」「給与」「転職」「報酬改定」が含まれると強い
- 新規サイトはまずロングテール（3語以上・月間100〜1,000）から始める
- ボリュームの大きいKWはピラー記事として後から狙う

## ⑤ SEOリサーチ手順（ライバルリサーチ）

```
STEP 1: 検索意図を3パターンで把握する
  └ Claudeへの依頼：
    「[KW]で検索するユーザーが知りたいことを3パターンに分類してください」

STEP 2: ライバル記事を3点で確認する（リベ大方式）
  └ ① タイトルにKWが含まれているか
  └ ② 記事内容がKWの検索意図を満たしているか
  └ ③ 自分の記事で差別化できる切り口・データがあるか
  └ ラッコの「見出し抽出」で上位20記事の構成を一括取得してClaudeに分析させる

STEP 3: ニーズの深掘り
  └ ラッコの「Q&A取得」でYahoo!知恵袋の質問を確認
  └ Xで「薬剤師 転職」「調剤薬局 給与」を検索してリアルな声を収集
  └ Claudeに要約させて記事に反映するポイントを抽出

STEP 4: E-E-A-T強化ポイントを決める
  └ 現役薬剤師ならではの一次情報・実体験・専門判断をどこに入れるか確認

STEP 5: 差別化ポイントを確定してPMが承認する
  └ 「この記事でしか読めない情報は何か」を1行で言えるまで絞り込む
```

## ⑥ 構成作成手順

### リード文の必須要素（リベ大方式）

リード文には以下の4要素を必ず入れる：
1. **想定読者の悩み**（「〜で悩んでいませんか？」）
2. **悩みの結論・解決策**（冒頭100字以内で先出し）
3. **解決できる理由・権威性**（現役薬剤師・収益構造データをもとに）
4. **読後のベネフィット**（「この記事を読めば〜が判断できます」）

### Claudeへの構成作成依頼テンプレート

```
【構成作成依頼】

メインKW: [KW]
フォルダ: [01_pillar / 02_articles / 03_data / 05_stock]
ターゲット読者: [病院 / 調剤薬局 / ドラッグストア] 薬剤師
読者の悩み: [具体的な悩みを記入]
記事の結論: [転職すべき / 残るべき / 様子見 / 知識習得]
参考データ: [調剤報酬改定の○○ / ○○社の決算 / など]
競合の見出し構成: [ラッコの見出し抽出結果を貼り付け]

出力:
- タイトル案 3つ（32文字以内・メインKW含む）
- ディスクリプション 2案（120文字以内）
- リード文の構成要素（4要素に沿って）
- H2見出し（5〜8個）
- 各H2のH3（2〜4個）
- 想定文字数
- 収益CTA設置箇所
- 内部リンク候補（設置箇所と紐づく記事名）
```

**構成のルール**:
- 結論を先に述べてから根拠を展開（PREP法: 結論→理由→具体例→結論）
- 比較表・数値データは積極的に使う
- 「転職すべきか・残るべきか・様子見か」の3択を明示するセクションを必ず入れる
- まとめ記事は詳細な説明を入れすぎず、詳細記事への誘導に徹する

## ⑦ 記事作成手順

```
STEP 1: PMが構成を承認する

STEP 2: Claudeに執筆を依頼する（付録Dのテンプレを使う）

STEP 3: 公開前チェックリストを確認する（下記参照）

STEP 4: PMが専門的事実確認・加筆修正を行う

STEP 5: パートナーがWPに入稿・装飾

STEP 6: PMが最終確認して公開
```

### 装飾ルール（リベ大方式）

記事の読みやすさのために以下を守る：
- **太字・赤字・マーカー**：目立たせたい部分に使う（多用しない）
- **箇条書き**：3つ以上の同列項目・長文が続く箇所で使う
- **吹き出し**：読者の疑問や著者の補足に使う
- **比較表**：職種別・サービス比較など数値を並べる際に使う
- **スマホ確認必須**：WP入稿後にスマホプレビューで文字詰まりを確認する

### 公開前チェックリスト

- [ ] 冒頭に「この記事でわかること」（箇条書き3〜5個）
- [ ] リード文に4要素（悩み・結論・理由・ベネフィット）が入っているか
- [ ] PREP法に沿った文章構成になっているか
- [ ] 著者の専門性・一次情報（現役薬剤師の視点）が含まれているか
- [ ] 調剤報酬改定の情報は最新年度のものか
- [ ] 比較表・数値データに出典が明記されているか
- [ ] 転職サイトCTAは記事の流れに合っているか（唐突でないか）
- [ ] スマホ表示で文字詰まりがないか確認したか
- [ ] メタディスクリプションが設定されているか
- [ ] 内部リンクは3〜7本設置されているか

## ⑧ 内部リンク設計手順

### トピッククラスター構造

```
【まとめ記事・ピラーページ（01_pillar）】
  ↑ 集客記事・詳細記事からリンクが集まる
  ↓ 各詳細記事へリンクを張る（本文中・自然な流れで）
  ※ まとめ記事は詳細な説明を書かず、詳細記事への入口に徹する

【集客記事・クラスターページ（02_articles / 05_stock）】
  ↑ データ記事・ニュース記事からリンクが来る
  ↓ まとめ記事・ピラーページへ必ずリンクを張る（相互リンク）
  ↓ 関連する他の集客記事にもリンクを張る（記事同士の連携）

【データ記事（03_data）】
  ↓ 関連するまとめ記事・集客記事へリンク
```

**内部リンクのルール**:
1. 1記事に内部リンクは3〜7本（多すぎない）
2. アンカーテキストはKWを含む自然な文章にする（「こちら」は使わない）
3. まとめ記事と詳細記事は**必ず相互リンク**する
4. 新記事を公開するたびに、既存記事から新記事へのリンクも追加する
5. ピラー記事は常に最多のリンクを受け取る状態を維持する

**内部リンク管理テーブル**（`03_data/_index.md`に随時更新）:

```
記事タイトル | URL | カテゴリ | リンク元 | リンク先
```

## ⑨ 収益導線設計手順

### 収益の流れ

```
検索流入記事（05_stock）← ロングテールKWで流入
  └→ データ・解説記事（03_data）← 収益構造・決算データ
       └→ まとめ記事・ピラー（01_pillar）← テーマ全体を整理
            └→ 転職判断記事（02_articles）← 転職すべきか判断
                 └→ キラーページ（02_articles）← 転職サイト比較
                      └→ 転職サイトCTA → 登録
```

### CTAの設置ルール

| 記事タイプ | CTA設置場所 | CTA内容 |
|---|---|---|
| 05_stock（流入記事） | 記事末尾のみ | 「給与が妥当か気になる方へ」ソフトCTA |
| 03_data（データ記事） | 記事末尾＋関連リンク | 「実際の転職相談はこちら」 |
| 02_articles（判断記事） | 記事中盤＋末尾 | 「転職メリットがある場合はこちら」 |
| 01_pillar（まとめ記事） | 複数箇所 | 転職サイト比較・ランキング形式のCTA |

**CTAの鉄則（転職煽り禁止）**:
- 「今すぐ転職しよう」は書かない
- 「あなたの給与は低すぎる」は書かない
- 「合理的に転職メリットがある場合は相談してみてください」のトーンを維持する
- 転職サイトは「情報収集ツール」として紹介する（「登録するだけでも市場価値がわかります」）

**想定アフィリエイト先（予定）**:
- マイナビ薬剤師
- ファルマスタッフ
- 薬キャリAGENT
- リクナビNEXT

## ⑪ リライト手順

リライトは記事を書くことと同様に重要な作業。50記事以上になってきた段階で本格的に取り組む。

### リライトの目的（優先順）
1. 検索順位を上げる（狙ったKWでの上位表示）
2. 回遊率を上げる（集客記事→まとめ記事→キラーページへの流れを改善）
3. CTA転換率を上げる（転職サイト登録につなげる）

### リライト手順

```
STEP 1: リライト候補を選ぶ
  └ サーチコンソールで「表示されているがクリックされていない記事」を探す
  └ GA4で「流入はあるが離脱率が高い記事」を探す
  └ 既に上位表示されている記事はリライトしない（順位が落ちるリスクがある）

STEP 2: 現状を確認する
  └ どのKWで検索されているか（サーチコンソール）
  └ KWの検索意図と現在の記事内容がズレていないか
  └ ライバル上位記事と比べて何が不足しているか

STEP 3: ライバルを再リサーチする
  └ ラッコの「見出し抽出」で現在の上位記事を再確認
  └ 自分の記事に不足しているH2・H3を特定する
  └ ライバルより強いコンテンツを追加できるか判断する

STEP 4: リライト内容を決めてClaudeに依頼する
  └ 追加すべきセクション・更新すべきデータを明示する
  └ CTAや内部リンクの改善も合わせて依頼する

STEP 5: 更新・公開後に効果測定する
  └ 最低1ヶ月様子を見てから効果を判断する
  └ 順位・クリック率・CVR（転職サイト誘導数）を確認する
```

**リライトの注意点**:
- 上位表示されている記事は触らない
- キーワードを無理に詰め込まない（ペナルティリスク）
- 内容が似た記事が増えたら統合を検討する（重複コンテンツ対策）
